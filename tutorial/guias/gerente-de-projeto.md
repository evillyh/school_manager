# Guia — Gerente de Projeto (PM)
← [Voltar ao README geral](../../README.md) · [Índice do tutorial](../README.md)

## Objetivo
Garantir que a turma entregue o **MVP** (Login/Perfis, Alunos, Turmas, Matrículas, Notas/Boletim) no prazo, removendo bloqueios e mantendo a integração entre as equipes.

## O que fazemos no MVP
- Priorizar as histórias da semana e **congelar escopo**.
- Conduzir ritos (daily, planning, review e retro).
- Agendar e proteger as **janelas de integração** (qua/sex).
- Acompanhar métricas simples: tarefas concluídas, bugs abertos/fechados.

## Entregáveis deste bimestre
- [ ] Backlog do MVP priorizado
- [ ] Calendário de integrações
- [ ] Ata de review com decisões
- [ ] Pitch para a demo final

## Definition of Done (PM)
- Ritos ocorreram (checklist marcado)
- Backlog atualizado e visível
- Integração da semana testada (smoke do QA)
- Decisões documentadas no quadro/guia

## Rotina da semana
1. **Quarta (começo)**: revisar backlog e prioridades com líderes  
2. **Quarta (final)**: integração + smoke test  
3. **Sexta**: integração + review/retro curtas

## Ferramentas
- Quadro Kanban, GitHub PRs, Postman (com o QA)

## Primeiras tarefas
- [ ] Publicar cronograma e metas semanais
- [ ] Abrir issues de alto nível por módulo
- [ ] Definir critério de aceite por história
